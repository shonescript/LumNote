# LumNote：不止是 Markdown 编辑器

LumNote 是一款基于 .NET 与 Avalonia 构建的跨平台 Markdown 工具，它把「写作编辑」「实时渲染」「版本协作」「内容导出」整合在同一个工作台中。  
你可以把它理解为：面向技术写作与知识沉淀场景的桌面级 Markdown 生产系统，而不是一个只负责预览的轻量工具。

## 一句话定位

一个内置自研渲染引擎与 Git 工作流的 Markdown 编辑器：从输入、排版、校对到版本管理与多格式发布，全链路在同一界面完成。

## 核心能力全景

- 实时编辑与预览：左侧编辑、右侧渲染，输入后自动刷新，支持预览区选择复制与缩放。
- 工作区与多标签：支持文件夹/多根工作区、文件树管理、多文档标签、最近文件与会话恢复。
- 全文搜索与跳转：面向工作区的跨文件搜索，支持结果分组、定位跳转与导航历史回退。
- 多格式导出：内置 HTML、PDF、PNG 长图导出，直接将当前文档发布为可分享内容。
- Git 一体化面板：状态查看、暂存/取消暂存、提交、分支切换、拉取推送、提交历史与双栏差异对比。
- 可配置主题样式：支持深浅主题与 Markdown 样式预设，界面与排版参数可持久化配置。

## 技术特点（为什么它“特别”）

- 纯 C# Markdown 解析器：核心解析逻辑自研实现，支持标题、列表、表格、任务列表、引用、代码块、脚注、链接图片、HTML 块等常见语法。
- 独立数学公式子系统：内置数学解析与渲染流程，支持行内与块级公式、分数/根式/上下标、常见数学命令与环境。
- 块级渲染架构：采用类似浏览器块级布局思路，布局与绘制分层，便于扩展且更稳健。
- 增量与可见区优化：围绕大文档场景做了可见区布局、滚动节流与区间计算优化，减少无效计算，提升长文档可用性。
- 导出与预览共用核心引擎：长图与 PDF 导出直接复用渲染能力，视觉结果更接近编辑时看到的内容。
- AOT/Trim 工程化策略：面向发布体积与启动部署优化，支持自包含交付，降低运行环境依赖。

## 体验细节（从“能用”到“好用”）

- 编辑区与预览区滚动联动，降低长文阅读/校对时的定位成本。
- 支持文档编码切换与重载，兼容不同来源文本文件。
- 自动保存策略可配置，遇到外部文件变化或缺失会提供可恢复交互。
- 图片可在标签页或独立窗口查看，链接支持点击打开。
- 快捷键、标签关闭确认、焦点回退/前进、工作区刷新等细节完整。

## 差异化价值

很多工具擅长“写”或“看”，LumNote 重点解决的是“生产闭环”：

1. 写作阶段：编辑、预览、搜索、样式调优在同屏完成。  
2. 协作阶段：Git 操作与提交历史不必切出到命令行。  
3. 交付阶段：一键导出 HTML/PDF/长图，快速进入分享与发布。  

这使它更像一套面向个人开发者、技术团队和内容创作者的文档工作台。

## 适用人群与场景

- 开发者：编写技术文档、设计说明、变更记录、发布说明。
- 团队协作：在版本管理语境下处理 Markdown 文档评审与演进。
- 内容创作者：输出教程、知识库、课程讲义并快速导出分发。
- 长文用户：面向章节较多、篇幅较长的结构化文档编辑与排版。
